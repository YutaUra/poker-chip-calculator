## Why

初回利用時にチップ構成を一から設定するのは面倒で、特に初心者は何を設定すべきか分からない（コールドスタート問題）。代表的なチップセットをビルトインで提供し、名前だけで選択できるようにすることで、初回体験を大幅に改善する。

## What Changes

- `BUILTIN_PRESETS` に代表的な3-5個のチップセットを追加する:
  - 「Standard (1/5/25/100/500)」- 汎用的なキャッシュゲーム構成
  - 「Tournament (100/500/1K/5K)」- トーナメント構成
  - 「Home Game (10/25/50/100)」- ホームゲーム構成
- PresetDialog でビルトインプリセットとユーザープリセットを視覚的に区別表示する（ビルトインは削除不可）
- ビルトインプリセットにはブラインド値も含め、選択時にチップ構成とブラインドの両方が一括設定される

## Capabilities

### New Capabilities

- `builtin-presets`: ビルトインプリセットのデータ定義と区別表示に関する仕様。シンプルな名称と直感的なチップ構成の提供、ビルトインとユーザープリセットの区別ロジックを包括する。

### Modified Capabilities

- `ui`: PresetDialog でビルトインプリセットとユーザープリセットをセクション分けして表示し、ビルトインは削除不可とする。

## Impact

- **変更ファイル**: `src/lib/presets.ts`（BUILTIN_PRESETS の定義変更）、`src/components/PresetDialog.tsx`（区別表示の強化）
- **依存関係**: 追加なし
- **ストレージ**: ビルトインプリセットはコード内定数のみ。ストレージには保存しない。

## Cross-Feature Dependencies

- **ante-and-m-ratio**: `ChipPreset` 型に `anteAmount?`, `anteUnit?`, `players?` をオプショナルとして予約しておくべき。ante-and-m-ratio 実装後にトーナメント系プリセットがアンティを含められるようになる
- **チュートリアル**: チュートリアル完了後にプリセット選択を促す導線設計が理想。チュートリアル最終ステップで「プリセットを選んで始めましょう」と案内するフローを検討
- **session-share**: URLパラメータで読み込んだチップ設定を「プリセットとして保存」するボタンの連携が自然な UX になる
- **chip-count-reset**: Chips ヘッダーに Reset ボタンと Presets ボタンが並ぶ → モバイルでの配置バランスに注意
