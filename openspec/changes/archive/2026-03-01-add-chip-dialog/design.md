## Context

現在の「add chip」は即座にデフォルト値（amount:1, unit:"1", color:"#6b7280"）でチップを追加する。ユーザーは追加後に ChipIcon をタップして編集ダイアログを開き、金額・単位・色を設定する。この2ステップを1ステップに統合する。

ChipIcon.tsx には既にチップ編集ダイアログ（金額入力 + カラーパレット）が実装されており、同じUI構成を再利用できる。

## Goals / Non-Goals

**Goals:**
- 「add chip」ボタンで設定ダイアログを表示し、金額・単位・色を設定してから追加する
- ChipIcon の編集ダイアログと同じ UI 部品を再利用する
- Cancel でチップは追加されない

**Non-Goals:**
- ChipIcon の既存編集ダイアログの変更
- チップの枚数（count）をダイアログで設定する（追加後にカウンターで操作する方が自然）

## Decisions

### 1. チップ編集UIを共通コンポーネントとして抽出する

**選択**: ChipIcon 内のダイアログ body（金額入力 + カラーパレット）を `ChipEditForm` として抽出し、ChipIcon と add chip ダイアログの両方で使う
**代替案**: add chip ダイアログに ChipIcon のコードをコピペする

**理由**:
- 金額入力 + カラーパレットの UI は完全に同じ。重複を避けることで保守性が向上する
- ChipIcon のテスト（24件）で検証済みの UI をそのまま活用できる

### 2. ダイアログは PokerChipCalculator 内で管理する

**選択**: `showAddChipDialog` state を PokerChipCalculator に追加し、ダイアログの開閉を管理する
**代替案**: add chip ボタン自体を Dialog コンポーネントにする

**理由**:
- 他のダイアログ（リセット確認、メモ編集）と同じパターンを踏襲する
- addChip ロジック（ID採番、ソート挿入）が PokerChipCalculator にあるため、同じスコープが自然

### 3. デフォルト値

**選択**: ダイアログの初期値は amount:100, unit:"1", color:"#ef4444"（赤）
**代替案**: 空欄/グレーのまま

**理由**:
- ポーカーで最も一般的なチップ単位は 100。空欄だと Save を押した時に 0 になるリスクがある
- 赤はポーカーチップの最も一般的な色

## Risks / Trade-offs

- **[ChipEditForm 抽出のリファクタリング]** → ChipIcon.tsx の既存テスト 24 件が壊れないよう、リファクタリングは段階的に行う。まず抽出して既存テストが通ることを確認し、その後 add chip ダイアログに統合する
