## Why

ユーザーが自身のプレイデータを持ち出せる手段がない。CSV/JSON エクスポートで外部ツール（Google スプレッドシート、PokerTracker 等）との連携を可能にし、JSON インポートでデバイス間のデータ移行やバックアップ復元を実現する。

## What Changes

- 現在のセッションのスナップショットデータを CSV 形式でエクスポートする機能を追加する
- 現在のセッションのスナップショットデータを JSON 形式でエクスポートする機能を追加する
- エクスポート内容: タイムスタンプ、BB 値、チップ総額、ブラインド額、ブラインド単位、メモ
- ダウンロードファイル生成（Blob + URL.createObjectURL）
- JSON ファイルからセッションデータをインポートする機能を追加する（ファイル選択 UI）

## Capabilities

### New Capabilities
- `data-export`: スタックデータのエクスポート（CSV/JSON）およびインポート（JSON）に関するすべての機能を包括する。ファイル生成、ダウンロード、ファイル読み込み、データバリデーションを含む。

### Modified Capabilities
- `ui`: Stack Graph セクションにエクスポート/インポートボタンを追加する。

## Impact

- **新規ファイル**: `src/lib/data-export.ts`（エクスポート/インポートロジック・バリデーション）
- **変更ファイル**: `PokerChipCalculator.tsx`（エクスポート/インポートボタンの統合、インポート時の状態更新）
- **ストレージ**: 変更なし（エクスポートはファイルダウンロード、インポートはファイル読み込み）
- **依存関係**: 追加なし（Blob API, URL.createObjectURL, FileReader はブラウザ標準 API）

## Cross-Feature Dependencies

- **multi-session-history**: アーカイブデータのエクスポート/インポートも対象にすべき。multi-session-history と同時に設計することで、エクスポート形式の一貫性を確保できる
- **StackSnapshot 型拡張**: ante-and-m-ratio 実装後の拡張フィールド（mValue, anteAmount）もエクスポート対象に含める必要がある。エクスポート形式のバージョニングを検討
- **session-share**: URLパラメータ経由のチップ設定読み込みと、JSON インポートによるセッションデータ読み込みは異なる機能だが、ユーザーの混乱を避けるために UI 上の動線を整理する必要がある
- **PokerChipCalculator.tsx 肥大化**: エクスポート/インポートボタンとダイアログの状態が追加される。Stack Graph セクションへの配置で既存 UI が混雑しないよう注意
- **チュートリアル**: パワーユーザー向け機能のためチュートリアルには含めない
