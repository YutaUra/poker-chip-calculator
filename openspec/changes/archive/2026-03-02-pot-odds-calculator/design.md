## Context

現在のアプリはスタック管理に特化しており、ハンド中のベッティング判断を支援する機能はない。ライブポーカーのプレイ中、ポットオッズの暗算は頻繁に必要であり、特にポットが複雑な金額になった場合にミスが起きやすい。

ポットオッズの定義:
- potOdds = callAmount / (potSize + callAmount)
- 必要エクイティ = potOdds（コールが+EVになるために手札に必要な勝率）

## Goals / Non-Goals

**Goals:**
- ポットサイズとコール額からポットオッズをパーセントで即座に計算・表示する
- 必要エクイティ（= ポットオッズ）を並べて表示し、コール判断の定量的根拠を提供する
- トグルでON/OFFを切り替え可能にし、不要な場合はUIを隠す
- 入力状態をsessionStorageに永続化する

**Non-Goals:**
- ハンドレンジに基づくエクイティ計算（モンテカルロシミュレーション等）
- インプライドオッズ（将来のストリートで追加で得られるチップ）の計算
- レイズサイズの推奨
- ポット/コール額のチップ自動連携（スタック計算とは独立した入力）

## Decisions

### 1. 計算ロジックの分離

`src/lib/pot-odds.ts` に独立モジュールとして実装する。

```typescript
interface PotOddsInput {
  potSize: number   // BB単位
  callAmount: number // BB単位
}

interface PotOddsResult {
  potOdds: number        // 0-100 のパーセント値
  requiredEquity: number // 0-100 のパーセント値（= potOdds）
}

function calculatePotOdds(input: PotOddsInput): PotOddsResult
```

potOdds と requiredEquity は数学的には同値だが、ポーカープレイヤーにとって「ポットオッズ」と「必要エクイティ」は異なる概念として認識されているため、両方を明示的に返す。

### 2. 入力のBB単位統一

ポットサイズ・コール額はBB単位で入力する。ライブポーカーでは「ポット5BB、相手が3BBベット」のようにBB基準で状況を把握するのが一般的。チップ額での入力は、このツールの主なユースケース（ライブゲームのクイック計算）には過剰。

**代替案**: チップ額入力もサポート → BB単位で十分。チップ額が必要なら自分で割り算すればよく、UIの複雑性に見合わない。

### 3. UI配置

Total Stack セクション（およびエフェクティブスタックセクション）の下に、独立した「Pot Odds」セクションとして配置する。トグルボタンでON/OFF切り替え（デフォルトOFF）。

ONにすると以下が展開される:
- Pot Size 入力（BB単位、数値入力）
- Call Amount 入力（BB単位、数値入力）
- 計算結果: Pot Odds と Required Equity のパーセント表示

### 4. 表示フォーマット

ポットオッズと必要エクイティは小数1位までのパーセント表示とする。例: "Pot Odds: 25.0%" / "Required Equity: 25.0%"。整数の場合は小数なしで表示する。例: "Pot Odds: 33%"。

### 5. 状態永続化

sessionStorage に以下のキーを追加:
- `pot-odds-enabled`: トグル状態（boolean）
- `pot-size-bb`: ポットサイズ（number）
- `call-amount-bb`: コール額（number）

## Risks / Trade-offs

- **[スタック管理との機能的距離]** → ポットオッズ計算はスタック管理と直接関係しないが、ポーカープレイヤーが同一セッション中に使う情報であり、同じアプリ内にあることに価値がある。トグルOFFをデフォルトにすることで、不要なユーザーには影響しない
- **[インプライドオッズ非対応]** → 実際のコール判断ではインプライドオッズも重要だが、計算に必要な前提（相手のレンジ、将来のベッティングアクション）が多すぎて自動化困難。ポットオッズのみに絞る
