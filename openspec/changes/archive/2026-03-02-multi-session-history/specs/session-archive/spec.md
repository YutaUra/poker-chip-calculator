## ADDED Requirements

### Requirement: アーカイブセッションのデータ構造を定義する

システム SHALL アーカイブセッションを以下の構造で表現する:
- `id`: 元の Session の ID（string）
- `startedAt`: セッション開始時刻（number, Unix ms）
- `endedAt`: セッション終了時刻（number, Unix ms）
- `snapshots`: 元の Session の StackSnapshot 配列
- `summary`: 事前計算済みのサマリー情報

サマリー情報は以下を含む:
- `snapshotCount`: スナップショット数
- `durationMs`: プレイ時間（endedAt - startedAt）
- `startBB`: 最初のスナップショットの BB 値
- `endBB`: 最後のスナップショットの BB 値
- `deltaBB`: endBB - startBB
- `peakBB`: セッション中の最大 BB 値
- `lastBlindAmount`: 最後のスナップショットのブラインド額
- `lastBlindUnit`: 最後のスナップショットのブラインド単位

#### Scenario: サマリーの計算が正しい

- **WHEN** スナップショットが [100BB, 150BB, 80BB] のセッションをアーカイブする場合
- **THEN** startBB は 100、endBB は 80、deltaBB は -20、peakBB は 150 となる

#### Scenario: スナップショットが1件のセッション

- **WHEN** スナップショットが1件のみのセッションをアーカイブする場合
- **THEN** startBB と endBB は同じ値となり、deltaBB は 0 となる

### Requirement: セッションリセット時にアーカイブ保存する

システム SHALL セッションリセット時に、現在のセッションをアーカイブとして保存するオプションを提供する。

#### Scenario: 保存してリセット

- **WHEN** スナップショットが1件以上存在する状態でリセットを実行し、「保存してリセット」を選択した場合
- **THEN** 現在のセッションが ArchivedSession に変換されて localStorage に保存される
- **THEN** 新しいセッションが開始される

#### Scenario: 保存せずリセット

- **WHEN** リセット実行時に「保存せずリセット」を選択した場合
- **THEN** 現在のセッションは保存されずに破棄される
- **THEN** 新しいセッションが開始される

#### Scenario: スナップショットが0件の場合

- **WHEN** スナップショットが0件の状態でリセットを実行する場合
- **THEN** アーカイブ保存の選択肢は表示されず、即座にリセットされる

### Requirement: アーカイブセッションを localStorage に永続化する

システム SHALL アーカイブセッションを localStorage の `session-archive` キーに JSON 配列として保存する。新しいセッションが配列の先頭に追加される。

#### Scenario: アーカイブの永続化

- **WHEN** セッションをアーカイブ保存した場合
- **THEN** localStorage の `session-archive` キーに即座に書き込まれる

#### Scenario: ページリロード後のアーカイブ復元

- **WHEN** ページをリロードした場合
- **THEN** localStorage からアーカイブセッションが復元され、履歴一覧に表示される

### Requirement: セッション履歴一覧を表示する

システム SHALL 過去のアーカイブセッションを一覧で表示する機能を提供する。各セッションは日付、プレイ時間、最終BB値、+/- BB を表示する。

#### Scenario: 履歴一覧の表示

- **WHEN** セッション履歴画面を開いた場合
- **THEN** アーカイブセッションが新しい順に一覧表示される
- **THEN** 各セッションに日付、プレイ時間、最終BB値、+/- BB が表示される

#### Scenario: アーカイブが0件の場合

- **WHEN** アーカイブセッションが0件の状態で履歴画面を開いた場合
- **THEN** 「まだセッション履歴がありません」のプレースホルダーが表示される

### Requirement: 通算サマリーを表示する

システム SHALL セッション履歴画面のヘッダーに通算成績のサマリーを表示する。

#### Scenario: 通算サマリーの内容

- **WHEN** アーカイブセッションが存在する場合
- **THEN** 総セッション数、通算 +/- BB、勝ちセッション数、負けセッション数が表示される

#### Scenario: 通算サマリーが0件の場合

- **WHEN** アーカイブセッションが0件の場合
- **THEN** 通算サマリーは表示されない

### Requirement: セッション詳細を表示する

システム SHALL 過去のセッションの詳細情報とグラフを表示する機能を提供する。

#### Scenario: 詳細表示

- **WHEN** 一覧からセッションを選択した場合
- **THEN** そのセッションの BB 推移グラフとチップ額推移グラフが表示される
- **THEN** グラフは閲覧専用で、記録の追加や削除はできない

### Requirement: アーカイブセッションを削除する

システム SHALL 個別のアーカイブセッションを削除する機能を提供する。

#### Scenario: セッションの削除

- **WHEN** アーカイブセッションの削除ボタンをタップした場合
- **THEN** 対象のセッションがアーカイブから削除される
- **THEN** localStorage が更新される
- **THEN** 通算サマリーが再計算される

### Requirement: アーカイブの容量を管理する

システム SHALL アーカイブセッションを最大50件に制限する。

#### Scenario: 50件を超えるアーカイブ

- **WHEN** アーカイブが50件の状態で新しいセッションをアーカイブ保存する場合
- **THEN** 最も古いセッションが自動的に削除される
- **THEN** 新しいセッションが追加される

#### Scenario: localStorage の書き込みエラー

- **WHEN** localStorage の容量制限により書き込みに失敗した場合
- **THEN** トースト通知でエラーを表示する
- **THEN** セッションリセットは実行される（アーカイブ保存のみ失敗）
