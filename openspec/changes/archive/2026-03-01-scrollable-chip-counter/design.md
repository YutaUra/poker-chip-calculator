## Context

チップ個数の入力が `<Input type="number">` のテキストフィールドで、モバイルではキーボードが起動し操作性が悪い。ユーザーはドラムロール型UIで直感的に数値を増減したいと要望している。

**重要なUX方針**: タップ→展開の2ステップではなく、数値部分がそのままインラインのドラムロールピッカーとして機能する。チップ行内に常にピッカーが表示されており、触れた瞬間からスクロール操作で値を変更できる。

**操作モデル（ジョグダイアル型）**: 数値エリアを押し込み、指を離さずに上下に移動すると値が変化する。押し込んだ位置（原点）から近ければゆっくり、遠ければ速くスクロールする。指を離すと停止する。

現在の実装: `src/components/PokerChipCalculator.tsx` の81-88行目にある `<Input>` コンポーネントを置き換える。

## Goals / Non-Goals

**Goals:**
- ジョグダイアル型の操作でチップ個数（0〜999）を選択可能にする
- 押し込み位置からの距離に応じた速度制御を実装する
- ドラムロール風の見た目（選択値 + 前後の数値）を表示する
- マウスホイール・キーボード操作にも対応する
- 既存の `updateChipCount` インターフェースとの互換性を維持する

**Non-Goals:**
- 1000以上の値への対応
- 水平方向の操作
- 外部ライブラリの追加

## Decisions

### 1. PointerEvent + requestAnimationFrame によるジョグダイアル実装

**選択**: PointerEvent で押し込み位置を原点として記録し、ポインタ移動量に応じた速度で rAF ループ内で値を増減する

**理由**: CSS scroll-snap ではこの「距離ベースの速度制御」は実現できない。PointerEvent はタッチ・マウス・ペンを統一的に扱え、`setPointerCapture` でコンポーネント外に指が出ても追従できる。

**代替案**:
- CSS scroll-snap → 距離ベースの速度制御が不可能。0〜999の範囲ではフリックでの移動も非現実的
- `react-scroll-picker` 等の外部ライブラリ → ジョグダイアル型のインタラクションを提供するものがない

### 2. 速度カーブ

**選択**: 押し込み位置からの距離（px）を速度（値/秒）に変換する。線形またはべき乗カーブで調整。

```
距離 0〜10px:  停止（デッドゾーン）
距離 10〜30px: 1〜3 値/秒（微調整）
距離 30〜80px: 3〜20 値/秒（通常）
距離 80px〜:   20〜100 値/秒（高速）
```

数値は実装時にチューニング。rAF ループ内で前フレームからの経過時間 × 速度で浮動小数の累積値を更新し、`Math.round()` で整数化して表示・コールバック。

### 3. 数値範囲は0〜999

**選択**: 0〜999の整数値を扱う。

**理由**: ハイステークスのゲームではチップ枚数が100を超えることがある。ジョグダイアル型の操作なら距離ベース加速で大きな値への移動もスムーズ。DOM には1000個のアイテムを並べる必要はなく、表示する3行分だけをレンダリングすればよい。

### 4. 表示は3行のみ動的レンダリング

**選択**: 現在値と前後1つの計3行だけをレンダリングし、値変化時にテキストを更新する

**理由**: 0〜999の全アイテムをDOMに並べる必要がない。ジョグダイアル操作では実際にスクロールするのではなく、値を動的に更新して数字がスクロールしているように見せる（CSS transform: translateY のアニメーション）。

### 5. コンポーネント構造

```
ScrollableCounter (新規)
├── コンテナ div (固定サイズ、overflow: hidden)
│   ├── 数値表示 (前の値) — opacity で薄く
│   ├── 数値表示 (現在値) — 強調
│   └── 数値表示 (次の値) — opacity で薄く
├── 上下のグラデーションオーバーレイ
└── 中央のハイライトバー
```

PointerEvent ハンドラ:
- `onPointerDown`: 原点Y座標を記録、`setPointerCapture`、rAF ループ開始
- `onPointerMove`: 現在Y座標を更新（rAF ループが速度計算に使用）
- `onPointerUp` / `onPointerCancel`: rAF ループ停止、キャプチャ解放

## Risks / Trade-offs

- **[速度チューニング]** 距離→速度カーブの感触は実機で調整が必要 → 初期実装後にパラメータを定数として切り出し、調整しやすくする
- **[アクセシビリティ]** ジョグダイアル操作はスクリーンリーダーで利用困難 → `role="spinbutton"` + キーボード操作（上下矢印）で代替操作を保証する
- **[レイアウト]** ピッカーの高さが現在の Input より大きい → 3行表示を各行コンパクト（24px程度）にし、全体で72px程度に収める
