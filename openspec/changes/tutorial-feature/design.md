## Context

ポーカーチップ計算機は ScrollableCounter（ドラッグ操作）、チップアイコンクリックによる編集ダイアログ、スタック記録など、初見では操作方法が分かりにくい UI がある。現在はオンボーディングが存在せず、ユーザーは自力で操作を発見する必要がある。

技術スタック: React 19 + TypeScript + Tailwind CSS v4 + shadcn/ui。外部チュートリアルライブラリ（react-joyride 等）は導入せず、自前実装で軽量に保つ。

## Goals / Non-Goals

**Goals:**
- 初回アクセス時にステップバイステップのチュートリアルを自動表示する
- チュートリアルはスキップ可能で、ヘルプボタンからいつでも再表示できる
- 完了バージョンを localStorage に永続化し、バージョン更新時は再表示する
- 既存コンポーネントへの影響を最小限に抑える

**Non-Goals:**
- インタラクティブなハンズオン（実際に操作させる形式）は対象外。説明テキスト＋ハイライトのみ
- 多言語対応は対象外（日本語のみ）
- チュートリアルのステップをユーザーがカスタマイズする機能

## Decisions

### 1. チュートリアルライブラリを使わず自前実装する

**選択**: CSS オーバーレイ + ポップオーバーの自前実装
**代替案**: react-joyride、intro.js
**理由**: 依存を増やしたくない。ステップ数が少なく（4-5ステップ）、shadcn/ui の Popover や Dialog を活用すれば十分な UX を実現できる。react-joyride はバンドルサイズが大きく（~50KB gzip）、このアプリのスケールには過剰。

### 2. ハイライト方式は CSS box-shadow + z-index で実装する

**選択**: 対象要素に高い z-index を設定し、背景にセミトランスペアレントなオーバーレイを配置する
**代替案**: SVG マスクによるスポットライト効果
**理由**: CSS のみで実装でき、対象要素のインタラクティビティを維持しやすい。SVG マスクは要素のリサイズ追従が複雑になる。

### 3. ステップ対象の特定に data 属性を使用する

**選択**: `data-tutorial="step-name"` 属性をチュートリアル対象要素に付与し、`querySelector` で参照する
**理由**: React の ref を各コンポーネントに追加するよりも、既存コンポーネントへの変更が少ない。テスト時の要素特定にも再利用できる。

### 4. ポップオーバーの配置は対象要素の位置に応じて動的に決定する

**選択**: 対象要素の `getBoundingClientRect()` を元に、ビューポート内に収まる方向（上/下）にポップオーバーを表示する
**理由**: モバイルファーストのアプリで、画面サイズが様々なため固定配置は不適切。

### 5. 状態管理は useState + localStorage（バージョン番号）で行う

**選択**: `useLocalStorage` (usehooks-ts) で `tutorial-version` キーに完了済みバージョン番号（number）を保存し、コード側の `TUTORIAL_VERSION` 定数と比較して表示を判定する。`useState` で現在のステップを管理する。
**代替案**: `tutorial-completed: boolean` フラグで管理する
**理由**: チュートリアル内容を更新した際にバージョン番号を上げるだけで再表示できる。boolean フラグでは「一度見たユーザーに新しい内容を見せる」手段がない。既にプロジェクトで `useLocalStorage` を使用しており、パターンが統一される。

表示判定ロジック:
- `localStorage の tutorial-version` が存在しない、または `TUTORIAL_VERSION` より小さい → 自動表示
- `localStorage の tutorial-version` が `TUTORIAL_VERSION` と同じ → 表示しない
- ヘルプボタン経由 → バージョンに関わらず表示

## Risks / Trade-offs

- **[リスク] 要素の位置が動的に変わる場合にポップオーバーがズレる** → `ResizeObserver` や scroll イベントでの再計算は初期実装では見送り。ステップ進行時に毎回 `getBoundingClientRect()` を呼ぶことで対応。
- **[リスク] スクロール位置によって対象要素が画面外にある** → ステップ遷移時に `scrollIntoView()` で対象要素を画面内にスクロールする。
- **[トレードオフ] 自前実装のため、アニメーションやアクセシビリティが不十分になりうる** → 初期リリースではシンプルなフェード遷移のみ。キーボード操作（Escape でスキップ、Enter/Space で次へ）は実装する。
